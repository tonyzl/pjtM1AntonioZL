# Tech Runbook (Demo)

- Kubernetes: aplicar rolling updates con maxUnavailable=0 para evitar downtime.
- Seguridad: rotar secretos cada 90 dias y usar secret manager centralizado.
- Observabilidad: logs estructurados + trazas distribuidas + alertas por SLO.
- CI/CD: test, security scan, deploy canario y rollback automatizado.
- Base de datos: usar migraciones idempotentes y backups diarios verificados.
